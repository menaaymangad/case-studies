# Nano City — Marketplace Mobile App Case Study

Nano City is a mobile marketplace app for product discovery and shopping flows. The work focused on improving high-traffic product browsing, integrating backend APIs, and polishing the customer-facing mobile experience.

Unlike SANOUKA or Storix, this case study intentionally stays higher-level because the public-safe evidence is the delivered app experience and portfolio notes rather than an open source repository.

## The problem

Marketplace apps are sensitive to speed and clarity. If product lists feel slow, search/browsing becomes frustrating. If shopping flows feel messy, users leave before taking action.

The main areas that needed attention were:

- product list loading speed;
- API integration reliability;
- customer-facing UI fixes;
- smoother browsing and product discovery;
- responsive mobile layout polish;
- store-ready delivery quality.

---

## My role

I contributed as a Flutter engineer responsible for product-facing improvements.

My responsibilities included:

- improving product-listing behavior;
- integrating REST API flows;

- working with Firebase-related app behavior;
- preparing app quality for Play Store delivery.

- Integrated REST API data into customer-facing shopping screens.
- Fixed UI issues affecting the browsing experience.
- Improved marketplace interaction polish on mobile.
- Supported Firebase-backed app behavior.
- Connected portfolio evidence to the live Play Store listing.

## What this project shows

Nano City is smaller as a case study than SANOUKA, but it is still useful portfolio evidence because it shows:

- working inside a product-heavy app;
- improving existing customer-facing flows;
- handling API-backed UI states;
- tuning performance where users feel it directly;
- shipping improvements to a real marketplace app.

For my portfolio, it supports the broader story: I can improve existing products, not only build new apps from zero.
